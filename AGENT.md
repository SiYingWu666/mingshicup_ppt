# 智能餐盘明石杯 PPT 协作说明

## 项目定位

本仓库用于协作迭代“明石杯智能应用类比赛讲解视频用PPT”。当前整合版不是三份原始PPT的简单拼接，而是按讲解视频逻辑统一叙事、统一版式、统一标题体系后的汇总稿。

最终交付文件：

- `output/智能餐盘_明石杯讲解视频_整合版.pptx`
- `output/智能餐盘_明石杯讲解视频_整合版_含目录章节.pptx`

原始材料保存在：

- `source/智能餐盘_机械结构设计部分.pptx`
- `source/视觉与联网部分_优化版.pptx`
- `source/智能餐盘_电控技术路线与创新点_V1(1).pptx`

## 页面结构

新版整合稿为 21 页，包含目录页和章节分隔页。章节顺序不要随意调整：

1. 封面：智能餐盘系统
2. 目录：四个章节总览
3. 系统总览：机械结构 + 传感采集 + AI识别 + 联网上传
4. 章节页：机械结构设计
5. 机械结构总览：多层结构、分区餐格、摄像头支架
6. 机械结构亮点：多层结构 / 滑槽安装 / 防水密封 / 功能集成
7. 精准称重结构：餐格 -> 垫块 -> 应变式传感器 -> 支撑层
8. 章节页：视觉识别与联网上传
9. 视觉识别平台：Titan Board + 摄像头 + NPU + LCD
10. 轻量化模型：YOLO-Fastest，从PC验证到端侧部署
11. 多菜品识别展示：30+菜品覆盖 + 识别Demo
12. 数据融合：菜品类别 + 分区重量 + 温度 -> 热量/营养记录
13. 联网上传与前端看板原型
14. 章节页：温度与重量传感器
15. 温度与重量传感器总览：为什么传感器是核心
16. 自制柔性温度传感器：PI基底 + Cu/CuNi热电偶 + 温区判断
17. 自制柔性重量传感器：Pt薄膜应变 + 多点补偿 + 偏心误差优化
18. 电控采集链路：AFE低噪声采集 + RA6M5滤波/标定/打包
19. 章节页：实物功能性展示
20. 实物功能展示：重量、温度、菜品识别、网页/屏幕展示
21. 总结：机械承载、传感采集、AI识别、云端记录形成闭环

## 内容取舍规则

- 机械部分只保留四个核心亮点：多层结构、滑槽安装、精准称重、功能集成。
- 视觉部分保留：Titan Board、YOLO-Fastest、30+菜品覆盖、识别Demo、前端看板原型。
- 电控部分只提炼为：自制柔性温度/重量传感器、低噪声采集、RA6M5处理、四层PCB/算法滤波。
- 不恢复源PPT中的资料来源脚注、过细工艺参数、薄膜表征细节、重复的用户端/商家端展开页。
- 每页保持一个主结论，正文控制在 3-5 个短要点，优先服务视频口播。

## 视觉规范

- 统一使用 16:9 宽屏。
- 保持深色标题栏 + 浅色内容区。
- 主色：科技蓝/青绿。
- 视觉识别相关使用绿色强调。
- 温度与重量传感相关使用橙色强调。
- 不要混用原始三份PPT的不同模板、页码样式和标题样式。
- 新增图片优先放入页面右侧主视觉区，避免堆满整页。

## 图片占位规范

缺少的实拍图、动图或截图统一用尖括号占位，后续替换时保留同一含义：

- `<温度传感器实物图>`
- `<重量传感器实物图>`
- `<实物称重演示动图>`
- `<温度检测演示动图>`
- `<菜品识别屏幕录像>`
- `<网页/屏幕展示截图>`
- `<前端看板截图>`

替换素材时只替换占位框内图片，不要改动页码、章节顺序和主标题。

## 后续迭代建议

- 优先补齐第 12、13、15 页的实物图或演示动图截图。
- 第 10 页前端看板如果有新版截图，可直接替换占位图。
- 如果比赛口播时间较短，优先压缩第 15-18 页电控内容，不建议删除目录页、章节页、机械或视觉闭环页。
